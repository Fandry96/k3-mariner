## 2024-06-25 - Caching Search API Calls in Long-Running Agents
**Learning:** The agent instance `MarinerSearchTool` processes multiple back-to-back queries across long-running tasks. Calling DuckDuckGo search recursively without a local cache incurs unnecessary latency (~200ms+ per identical query) and risks hitting rate limits. Standard `@functools.lru_cache` cannot be directly applied to instance methods without causing memory leaks since it caches the `self` reference indefinitely.
**Action:** Implemented a bounded instance-level dictionary (`self._cache = {}`) inside the tool's `__init__` and `forward` logic. This reduces subsequent identical query times to <1ms and safely bounds memory by `.clear()`ing when it exceeds 50 entries, providing a safe, lightweight, dependency-free local caching strategy for stateful agent objects.
